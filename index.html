ما هي الأتمتة؟

الأتمتة هي جعل الآلات تؤدي المهام المتكررة بدلاً منك. فكّر بها كـ "مساعد لا يتعب ولا ينسى".

المثال الكلاسيكي

عندما يملأ شخص نموذجاً على موقعك، تذهب البيانات تلقائياً لـ Google Sheets، ويُرسَل إيميل ترحيب، وتُضاف مهمة في Notion — كل ذلك بدون أي تدخل منك.

لماذا الأتمتة؟

الموظف يعمل 8 ساعات. الأتمتة تعمل 24 ساعة، 7 أيام، بدون أخطاء، وبنفس الدقة في كل مرة. النتيجة: إنتاجية أعلى وتكاليف أقل.

العقد (Nodes)

العقدة هي اللبنة الأساسية في n8n. كل عقدة تؤدي مهمة واحدة محددة. الـ Workflow = مجموعة عقد متصلة.

Trigger Node

نقطة البداية — يبدأ سير العمل

المحرّك الأول

مثل زر التشغيل. يستجيب لحدث: وصول إيميل، ضغطة زر، جدول زمني، Webhook. كل workflow يبدأ بـ Trigger واحد.

Action Node

المنفّذ — يقوم بعمل محدد

المنفّذ الفعلي

يرسل إيميل، يكتب في Spreadsheet، يستدعي API، يعالج البيانات. معظم العقد في الـ Workflow هي Action.

Logic Node

المفكّر — يتحكم في التدفق

عقل الـ Workflow

IF/Else · Switch · Merge · Filter. يوجّه البيانات بناءً على شروط. بدونها الـ Workflow يسير في خط مستقيم فقط.

التوصيلات (Connections)

التوصيلات هي الأسلاك التي تربط العقد ببعضها وتنقل البيانات بينها. البيانات تتدفق من اليمين إلى اليسار.

كيف تتدفق البيانات

كل عقدة تُخرج JSON object يحتوي على البيانات. العقدة التالية تستقبله وتعمل عليه. البيانات تتراكم وتتحول في كل خطوة.

التفرع والدمج

يمكن لعقدة واحدة أن تُغذي أكثر من عقدة (تفرع)، أو أن تُدمج مسارات متعددة في مسار واحد (Merge Node).

المحفزات (Triggers)

المحفّز هو الحدث الذي يوقظ الـ Workflow. بدون محفّز، لا شيء يعمل تلقائياً.

Schedule Trigger

يعمل في أوقات محددة مسبقاً. مثل: كل يوم الساعة 8 صباحاً، أو كل ساعة، أو في أول يوم من كل شهر. مثالي للتقارير الدورية.

Webhook Trigger

يستقبل طلبات HTTP من تطبيقات خارجية. عندما يحدث شيء في تطبيق آخر يرسل إشعاراً لـ n8n فيبدأ الـ Workflow فوراً.

Email Trigger

يراقب صندوق البريد الوارد. عند وصول إيميل جديد يُشغّل الـ Workflow تلقائياً. مفيد لمعالجة الطلبات والاستفسارات.

Chat Trigger

يستقبل رسائل من المستخدمين. يُستخدم لبناء chatbots وواجهات دردشة مرتبطة بـ AI Agent. الأساس لبناء المساعدين الذكيين.

التعبيرات (Expressions)

التعبيرات تجعل الـ Workflow ديناميكياً. بدلاً من قيم ثابتة، تستخدم بيانات حقيقية من العقد السابقة.

الذكاء الاصطناعي في n8n

n8n يدمج الذكاء الاصطناعي مباشرة في سير العمل. من الردود البسيطة إلى الوكلاء المستقلين القادرين على التخطيط والتنفيذ.

نماذج اللغة (LLMs)

GPT-4 · Claude · Gemini · Llama. n8n يتصل بها عبر API ويستخدمها لتلخيص وترجمة وتوليد المحتوى داخل الـ Workflow.

وكلاء AI

الوكيل لا يردّ فقط، بل يفكّر ويختار الأدوات المناسبة لإنجاز المهمة. يمكنه البحث في الويب، قراءة ملفات، كتابة في قواعد البيانات.

الذاكرة (Memory)

تمكّن الوكيل من تذكّر سياق المحادثة. بدونها كل رسالة معزولة. مع الذاكرة يتذكر ما قيل قبله ويبني عليه.

الأدوات (Tools)

كل أداة هي شيء يمكن للوكيل فعله: البحث، إرسال إيميل، قراءة Spreadsheet. الوكيل يختار الأداة المناسبة بنفسه.

عقدة AI Agent

عقدة AI Agent هي قلب بناء الوكلاء في n8n. تحتاج إلى توصيل مكونات أساسية لكي تعمل بشكل صحيح.

System Prompt — الشخصية والتعليمات

تكتب فيه شخصية الوكيل وحدوده وأسلوبه. مثال: أنت مساعد خدمة عملاء محترف لشركة X، ترد باللغة العربية فقط.

الأدوات — قدرات التنفيذ

كل أداة تُضاف كـ sub-node: HTTP Request لاستدعاء APIs، Google Sheets للقراءة والكتابة، Code لتشغيل JavaScript مخصص.

بناء وكيل AI من الصفر

دليل خطوة بخطوة لبناء مساعد دردشة ذكي يتذكر المحادثة ويرد بذكاء.

١

أضف Chat Trigger

ابحث عن "Chat Trigger" في قائمة العقد وأضفه. هذه نقطة البداية التي تستقبل رسائل المستخدمين. اضغط "Test" لفتح نافذة الدردشة.

٢

أضف AI Agent Node

اضغط "+" بعد الـ Trigger، ابحث عن "AI Agent"، أضفه. ستجد ثلاثة موصلات في الأسفل: Chat Model، Memory، Tools.

٣

اربط Chat Model

من الموصل الأول، أضف "OpenAI Chat Model"، أضف Credentials الخاصة بـ OpenAI API Key، اختر نموذج gpt-4o.

٤

أضف الذاكرة

من الموصل الثاني، أضف "Simple Memory". هذا يمكّن الوكيل من تذكّر آخر 10 رسائل في المحادثة تلقائياً.

٥

اكتب System Prompt

في إعدادات AI Agent، أضف System Prompt يصف شخصية الوكيل ومهامه. كلما كان أكثر تفصيلاً كان الوكيل أدق.

٦

جرّب وأطلق!

اضغط "Chat" في الـ Trigger لفتح نافذة الاختبار. تحدّث مع الوكيل، راقب كيف يفكّر ويرد. فعّل الـ Workflow عند الرضا.
